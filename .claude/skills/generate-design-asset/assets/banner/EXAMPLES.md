## Glass Object Technical Examples

### Example 1: "The Agent Endgame" (Actual Post)

**Post Context**:
- **Category**: AI Engineering
- **Tags**: Custom agents, agent engineering, system prompts, agent architecture
- **Tone**: Advanced, architectural

**Style Selection**: Glass Object Technical
**Rationale**: Represents sophisticated custom agent architecture with emphasis on technical elegance and production-grade systems.

**Filled Prompt**:
```
A photorealistic, highly detailed image featuring a custom AI agent processing unit rendered in clear, highly polished transparent glass or crystal material. The object has distinct thickness and dimensional depth, with the iconic shape of a modular processor chip or sophisticated control system—presenting a central processing core with radiating connection pathways, input/output nodes arranged in geometric patterns, and layered internal circuitry structures in simplified yet extremely precise geometric structures, making it instantly recognizable without any patterns. All edges are treated with rounded chamfers and smooth curved surfaces, creating elegant refraction effects under light. The object is placed slightly tilted, as if floating above a clean, flawless, seamless pale beige studio background.

Lighting is bright, clean professional studio light, focusing on highlighting the transparency, specular reflection and refraction characteristics of the glass material. Sharp and delicate highlights appear on the object's chamfers, connection pathway edges and core processor ring, highlighting the crystal texture and luxurious vision. Subtle refraction, light bending and local distortion effects are produced when light penetrates the interior of the glass body, especially obvious in thickness variation areas and internal circuit layer details, greatly enhancing realism and visual impact. A soft, diffuse shadow falls below and slightly behind the object, giving the picture a sense of groundedness without destroying the minimalist temperament.

The overall aesthetic style is minimalist, modern, and clean, presenting the visual effect of high-end product photography and concept art rendering. The focus of the picture is completely on the crystal clear material performance and precise geometric shape of the glass AI agent processing unit. The image as a whole is high-key and shallow depth of field processing, keeping the object in absolute sharp focus, while the background is softly blurred, thereby maximizing the subject.

Technical specifications: 2560x1440 resolution, 16:9 aspect ratio optimized for blog header display.
```

**Expected Visual Description**:
A pristine glass processor chip hovers in clean space. Light refracts through transparent layers showing internal circuit pathways like frozen lightning. Edges catch brilliant highlights. The crystal structure suggests both precision engineering and elegant abstraction—perfect for representing the sophistication of custom agent systems. Background fades to soft neutral, keeping full focus on the floating technical object.

## Isometric Technical Diagram Examples

### Example 3: "You're Using Sub-Agents Wrong" (Actual Post)

**Post Context**:
- **Category**: AI Development
- **Tags**: Sub-agents, workflow optimization, context management, ai architecture
- **Tone**: Provocative but instructional

**Style Selection**: Isometric Technical Diagram
**Rationale**: Shows the correct research-focused pattern vs. wrong implementation-delegation pattern through spatial workflow visualization.

**Filled Prompt**:
```
Draw a hand-drawn isometric technical diagram of a multi-agent research workflow with file-based context sharing. The illustration uses clean, precise line art in a technical drawing style with consistent line weight and professional clarity. The diagram shows parent agent, specialized research sub-agents, shared file system, and final implementation output arranged in three-dimensional space using isometric projection (30-degree angles) to display spatial relationships and hierarchical structure.

FOREGROUND LAYER: Primary parent agent positioned in the foreground as a labeled box with bold outline (2.5pt line weight) showing "Parent Agent: Implementation". Connected to it is a document icon labeled "context.md" representing the shared file system. These elements are drawn with slightly thicker lines and include small labels with technical terminology.

MIDGROUND LAYER: Three specialized research sub-agents positioned in the middle ground, each represented as smaller boxes labeled "Research Agent: Vercel", "Research Agent: Stripe", and "Research Agent: Supabase". Arrows flow from each research agent TO the context.md file (not the reverse), showing information transfer. These elements use medium line weight (1.5pt) and show process flow with directional arrows.

BACKGROUND LAYER: Faded "incorrect pattern" ghost diagram in deep background showing the WRONG approach where parent delegates implementation to sub-agents. This uses lighter line weight (0.75pt) with red X overlay, providing contrast context without overwhelming the primary correct pattern.

Annotation style: Clean sans-serif labels positioned near components using leader lines. Labels use Title Case for agent types ("Parent Agent", "Research Agent") and lowercase for file references ("context.md", "shared state"). Key insight callout: "Research ONLY - No Implementation" near sub-agents.

Color palette: Black linework with cyan accent highlights for active data flow arrows, orange highlights for the correct pattern's key components, and red accent for the incorrect pattern in background.

Background: Clean white technical drawing background with subtle grid pattern (very light gray, 0.25pt lines) suggesting structured methodology.

Lighting and depth: Soft directional lighting from top-left creates subtle shadows beneath and behind agent boxes, establishing depth hierarchy without harsh shadows—maintaining technical drawing clarity.

Technical specifications: 2560x1440 resolution, 16:9 aspect ratio, optimized for blog header with clear readability at multiple scales.
```

**Expected Visual Description**:
Clean isometric view showing the correct agent pattern. Parent agent sits in foreground connected to a central context file. Three research sub-agents in midground each send arrows TO the context file (not receiving from it). The spatial arrangement makes the pattern obvious: research flows inward, implementation flows from shared context. In the background, a ghosted "wrong way" diagram shows the anti-pattern with a red X. Cyan data-flow arrows and orange highlights guide the eye to the correct approach.

Left-to-right workflow showing files dropping into an input folder, a watcher service monitoring it, and parallel agents processing different file types. Color-coded arrows (green for active processing) connect each stage. Output folders in the background show the final destinations. Title block: Traditional blueprint title block in bottom-right corner containing:

## Style Comparison: Same Post, Different Styles

### Post: "Context Engineering for AI Agents"

Demonstrating how the same post title would look across different banner styles.

---

#### Version A: Glass Object Technical

**Visual**: Transparent glass file system or context container, showing internal layered structure representing different context zones (system prompt layer, tool layer, conversation layer). Light refracts through the layers. Minimalist, sophisticated.

**Rationale**: Emphasizes the architectural nature of context engineering—building structured, layered systems.

---

#### Version B: Isometric Technical Diagram

**Visual**: Isometric view of context flow: user input → context retrieval → agent processing → output. Shows retrieval mechanisms, caching layers, and progressive disclosure patterns as spatial workflow.

**Rationale**: Shows the process and methodology of context engineering spatially.

---

#### Version C: Technical Blueprint

**Visual**: Blueprint schematic showing context window as measured bar with sections labeled: system (8%), tools (12%), conversation (50%), available (30%). Callouts show token counts and optimization strategies.

**Rationale**: Data-driven view emphasizing quantitative approach to context optimization.

---

#### Version D: Newspaper Front Page

**Visual**: Headline: "CONTEXT ENGINEERING: THE SECRET TO 10X AGENT PERFORMANCE". Photo shows before/after comparison of agent performance with proper context management.

**Rationale**: If the post takes a provocative stance about how most engineers ignore context engineering to their detriment.

---

#### Version E: Enhanced Glitch Corruption

**Visual**: Corrupted context window visualization with glitch effects showing "context overflow" or "token bleeding" through digital corruption aesthetic. Holographic layers represent different context zones.

**Rationale**: Universal fallback that works when content type is unclear, or if the post has a slightly edgy/modern take.

---

**Recommendation Logic**:
- If post is architectural/framework-focused → **Glass Object** (A)
- If post is process/methodology tutorial → **Isometric** (B)
