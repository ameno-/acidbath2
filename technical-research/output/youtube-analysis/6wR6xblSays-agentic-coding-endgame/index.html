Key Insights

  • System Prompts Are Foundational - System prompts completely redefine agents, transforming existing models into entirely new specialized computational products. They multiply the effect of every user prompt interaction.
  • Agent Progression Model - Agent engineering follows predictable progression: (1) better agents through prompt engineering, (2) more agents to scale compute, then (3) custom specialized agents for domain-specific problems.
  • Context Window Efficiency - Out-of-box agents carry 15+ unnecessary tools that consume context window space. Custom agents solve this by providing only needed tools, critical as systems scale.
  • Multi-Agent Orchestration - Multi-agent systems enable complex workflows like software development lifecycle automation by distributing specialized tasks across purpose-built agents.
  • Custom Agents Unlock Value - Custom agents solve organization-specific problems and edge cases that general-purpose tools cannot address effectively.
  • Agent Perspective Adoption - Monitoring what agents do and understanding their decision-making is essential for debugging, optimization, and system management.
  • Tool Integration Bridge - Custom tools bridge deterministic programming with conversational AI, combining predictable operations with intelligent reasoning.
  • Model Selection Strategy - Avoid over-engineering simple problems with expensive models. Use Claude Haiku for simple tasks and reserve powerful models for complex reasoning.
  • Self-Modifying Systems - Agent workflows can operate on themselves, creating self-modifying systems with real-time updates and autonomous improvements.
  • Deployment Focus - Effective agent deployment targets repetitive, high-value workflows where automation provides significant productivity multipliers.

Code Examples

Example 1: System Prompt Override (Pong Agent) 
from claude_code_sdk import CloudCodeOptions

def load_system_prompt():
    return """
    You are a pong agent.
    Always respond exactly with "pong".
    """

options = CloudCodeOptions(
    system_prompt=load_system_prompt()
)
Example 2: Custom Tool Definition (Echo Agent) 
@tool
def echo_tool(args: dict) -> str:
    """Echo text with transformations like reverse, uppercase, repeat"""
    text = args.get("text", "")
    reverse = args.get("reverse", False)
    uppercase = args.get("uppercase", False)
    repeat = args.get("repeat", 1)

    if reverse:
        text = text[::-1]
    if uppercase:
        text = text.upper()

    result = (text + " ") * repeat
    return result.strip()
Example 3: In-Memory MCP Server Creation 
from claude_code_sdk import create_sdk_mcp_server

mcp_server = create_sdk_mcp_server([echo_tool])

options = CloudCodeOptions(
    model="claude-3-haiku-20240307",
    mcp_servers=[mcp_server]
)
Example 4: Client vs Query Usage Pattern 
# For one-off prompts
result = query(user_prompt, options)

# For continuous conversations
client = ClaudeSDKClient(options)
response1 = client.query(user_prompt)
response2 = client.query(follow_up_prompt)  # Maintains conversation context
Example 5: Multi-Agent Orchestration Structure 
class MicroSDLCAgent:
    def __init__(self):
        self.planner_agent = self.create_agent("planner")
        self.builder_agent = self.create_agent("builder")
        self.reviewer_agent = self.create_agent("reviewer")
        self.shipper_agent = self.create_agent("shipper")

    def process_task(self, task):
        plan = self.planner_agent.query(task)
        implementation = self.builder_agent.query(plan)
        review = self.reviewer_agent.query(implementation)
        return self.shipper_agent.query(review)
