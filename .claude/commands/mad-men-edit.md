# Mad Men Edit Command

Transform weak copy into conversion-focused content using the Mad Men Copywriter skill.

## Usage

```bash
/mad-men-edit {file_path}
```

**Example**:
```bash
/mad-men-edit content/landing-pages/product-launch.md
```

The `/mad-men-edit` command analyzes content for conversion opportunities and generates aggressive, results-driven edit suggestions using Mad Men copywriting principles.

## Focus Areas

Mad Men Copywriter prioritizes these improvements:

### 1. Headlines & Hooks
**Looks for**: Generic announcements, weak openings, no pain/value

**Transforms to**: Pain-first hooks, attention grabs, immediate value

**Example transformation**:
```
‚ùå "Introducing Our New Analytics Platform"
‚úÖ "Stop Making $1M Decisions with Gut Feelings"
```

### 2. CTAs (Calls to Action)
**Looks for**: Generic actions ("learn more", "click here"), no benefit

**Transforms to**: Benefit-driven CTAs with specific value

**Example transformation**:
```
‚ùå "Click here to learn more"
‚úÖ "See Your 40% Cost Savings - Free Calculator Inside"
```

### 3. Value Propositions
**Looks for**: Vague promises, corporate speak, buried benefits

**Transforms to**: Specific, quantified outcomes

**Example transformation**:
```
‚ùå "We help companies improve efficiency"
‚úÖ "Ship 3x faster. Cut costs 40%. Same team, better tools."
```

### 4. Feature Descriptions
**Looks for**: Feature lists without benefits, technical specs without context

**Transforms to**: Outcome promises showing what customer achieves

**Example transformation**:
```
‚ùå "Real-time dashboards with custom reports"
‚úÖ "See problems the second they happen. Build reports your CEO reads."
```

## Anti-Pattern Checks

The Mad Men skill automatically avoids these anti-patterns:

### ‚úÖ Preserves Technical Sections
- Code blocks remain exact
- Benchmarks and data tables preserved
- Technical definitions maintain accuracy
- Configuration examples unchanged

### ‚úÖ No Auto-Modification
- Command only outputs suggestions
- Original file never written
- User maintains full editorial control
- Suggestions can be rejected or modified

### ‚úÖ Limits Suggestions
- Focuses on 5-8 high-impact changes
- Prioritizes hooks, CTAs, value props
- Doesn't suggest trivial tweaks
- Quality over quantity

### ‚úÖ Provides Specific Rationale
- Every suggestion explains conversion principle
- Cites frameworks (PAS, AIDA, etc.)
- Shows why edit improves conversion
- Avoids vague "makes it better"

## Integration with Other Skills

### Mad Men vs Master Copywriter
- **Mad Men**: Immediate conversion, aggressive tone
- **Master**: Long-term persuasion, authority building
- **Use Mad Men when**: Primary goal is signup/purchase/click
- **Use Master when**: Building trust and thought leadership

### Mad Men vs Ameno Voice
- **Mad Men**: Sales/marketing content conversion focus
- **Ameno**: Technical content simplification
- **Use Mad Men when**: Selling a product or service
- **Use Ameno when**: Explaining technical concepts

## Command Implementation

When you invoke `/mad-men-edit`, here's what happens:

### Step 1: Read Content
```markdown
- Use Read tool to load file
- Verify file exists and is readable
- No modifications occur
```

### Step 2: Section Classification
```markdown
- Identify section boundaries (headers, paragraphs)
- Classify each section by type:
  - introduction, concept-explanation, failure-mode
  - takeaway, cta, technical, navigation
- Mark edit candidates vs preserve sections
```

### Step 3: Conversion Analysis
```markdown
For each editable section:
- Evaluate against Mad Men checklist:
  - Hook strength (pain-first? attention grab?)
  - Value deployment (benefit by word 50?)
  - CTA clarity (specific action + benefit?)
  - Weak sentence presence (hedging, passive voice)
  - Scannability (short paragraphs, bullets)

- If weaknesses found ‚Üí Generate suggestion
- If already strong ‚Üí Skip or minimal tweak
```

### Step 4: Generate Suggestions
```markdown
For each conversion opportunity:
- Extract exact original content (Before)
- Apply Mad Men transformation (After)
- Write specific rationale (framework + principle + impact)
- Format as standard suggest_edits template
```

### Step 5: Output Results
```markdown
- Summary of total suggestions
- Individual edit suggestions with before/after
- Preserved sections documentation
- Application notes and next steps
- NO file writing occurs
```

## Mad Men Checklist (Used During Analysis)

When analyzing content, the skill checks:

- [ ] **Pain-first hook** - Does intro lead with what hurts?
- [ ] **Value by word 50** - Core benefit visible in first 2 sentences?
- [ ] **Weak sentences eliminated** - Every sentence earns its place?
- [ ] **Active voice dominant** - "You get X" not "X is provided"?
- [ ] **Specific benefits** - Concrete outcomes not vague features?
- [ ] **Scannable** - Short paragraphs, bullets, bold?
- [ ] **Emotion + Logic + Proof** - All three elements present?
- [ ] **Strong CTA** - Clear action with specific benefit?
- [ ] **Urgency present** - Reason to act now not later?
- [ ] **Technical sections preserved** - Code/configs/benchmarks accurate?

Any failed checklist item is a candidate for editing.

## Troubleshooting

### "No suggestions generated"
**Possible reasons**:
- Content is already conversion-optimized
- Content is pure technical (no conversion opportunities)
- File path incorrect

**Solution**: Verify file path, ensure content has marketing/sales elements

### "Too many suggestions"
**Should not happen** - Skill limits to 5-8 high-impact suggestions

**If it does**: Report as bug - skill should be selective

### "Suggestions don't fit my brand voice"
**Expected** - Mad Men is aggressive by design

**Solution**:
- Use "Apply with modifications" option
- Tone down aggression while keeping principle
- Consider `/master-copy-edit` for softer persuasion

### "Technical sections got edited"
**Should not happen** - Skill preserves code/benchmarks

**If it does**: Report as bug - technical accuracy is guaranteed
