üí° Key Insights

  • Sub-Agents Excel as Researchers, Not Implementers
    Implementation-focused sub-agents fail due to context isolation; research-focused agents provide valuable summaries for parent agent execution.
  • Context Engineering Saves 80% of Tokens
    File-based context management reduces token consumption from pre-work 80% usage to just hundreds of tokens per research task, preventing conversation compaction.
  • Specialized Service Agents Beat Generic Approaches
    Domain-specific agents (Vercel SDK, Stripe, Shadcn, Supabase, Tailwind) with embedded documentation and MCP tools dramatically outperform generic agents.
  • File System = Ultimate Context Management
    Using the file system as context storage enables persistent information sharing across agents without token bloat.
  • Parent Agents Need Full Implementation Context
    When sub-agents do implementation, parent agents lose visibility into changes, making debugging impossible.
  • Research Phases Improve Code Quality
    Planning and research before implementation leads to better architectural decisions and higher-quality code.
  • Token Optimization > Speed Initially
    Proper context management prevents performance degradation from conversation compaction, which is more important than raw execution speed.
  • Documentation Retrieval > Conversation History
    Storing information in retrievable files beats keeping everything in conversation history for long-running tasks.
  • Agent Orchestration Requires Explicit Context Files
    Successful multi-agent coordination depends on shared context files that all agents read and update.
  • Implementation Delegation Needs Information Architecture
    Without proper context sharing infrastructure, delegating implementation to sub-agents leads to failure.

üéØ Recommendations

Strategic Recommendations

  • Focus sub-agents on research and planning tasks, not direct code implementation
  • Create specialized sub-agents for each service provider you frequently use
  • Use file-based context management instead of conversation history to prevent token bloat
  • Maintain centralized project context files that all agents read before starting work
  • Design clear goals for sub-agents that explicitly exclude direct implementation

Tactical Recommendations

  • Include service-specific documentation directly in system prompts
  • Structure sub-agent communication with specific output formats
  • Use MCP tools to retrieve relevant components and examples
  • Implement background sessions for long-running tasks
  • Create migration guides for service updates

Implementation Recommendations

  • Test sub-agents thoroughly in production environments before wide adoption
  • Join community sessions to learn evolving best practices
  • Delegate tasks with specific context file references
  • Update context files after completing tasks
  • Embed specialized knowledge through documentation rather than general capabilities
